# 시니어 데이터 직무로 가는 길

> Data Science Career 101 시리즈 (10/10)

<!-- a-grade-intro:begin -->

**핵심 질문**: *시니어 데이터 직무* 는 *무엇* 이 *다른가요*?

> *문제 정의*, *영향 측정*, *동료 증폭*, *시스템*, *지속*.

<!-- a-grade-intro:end -->

## 이 글에서 배울 것

- *시니어* 의 *정의*
- *문제 정의* 능력
- *영향 측정*
- *동료 증폭*
- *시스템 사고*

## 왜 중요한가

*시니어* 는 *직급* 이 *아니라* *행동* 입니다.

## 개념 한눈에 보기

```mermaid
flowchart LR
    J[Junior] --> M[Mid]
    M --> S[Senior]
    S --> L[Lead]
```

## 핵심 용어 정리

- **scope**: *책임 범위*.
- **impact**: *영향*.
- **multiplier**: *증폭자*.
- **systems thinking**: *시스템 사고*.
- **mentorship**: *멘토십*.

## Before/After

**Before**: "*티켓* 만 *처리*."

**After**: "*문제* 를 *정의* 하고 *측정* 으로 *제안*."

## 실습: 5가지 시니어 행동

### 1단계 — 문제 정의

```text
"증상"이 아니라 "문제"를 적는다.
```

### 2단계 — 영향 측정

```markdown
- 가설
- 측정 계획
- 결과
```

### 3단계 — 멘토링

```text
- 월 1회 1:1
- 멘티가 의제 정한다
```

### 4단계 — 의사결정 기록

```text
- 결정 / 대안 / 근거 / 영향
```

### 5단계 — 시스템 사고

```text
- 데이터 → 모델 → 의사결정 → 운영 → 회고
```

## 이 코드에서 주목할 점

- *문제 정의* 가 *질* 입니다.
- *측정* 이 *증명* 입니다.
- *동료* 가 *증폭* 입니다.

## 자주 하는 실수 5가지

1. ***기술* 만 *판다*.**
2. ***영향* 측정 *없음*.**
3. ***멘토링* 을 *미룬다*.**
4. ***결정* 기록 *없음*.**
5. ***시스템* 으로 *생각* *안* *함*.**

## 실무에서는 이렇게 쓰입니다

빅테크 *시니어 트랙* 은 *Staff*, *Principal* 로 *이어집니다*.

## 시니어 엔지니어는 이렇게 생각합니다

- *문제 정의* 우선.
- *영향* 측정.
- *동료* 증폭.
- *결정* 기록.
- *시스템* 관점.

## 체크리스트

- [ ] *문제 정의* 양식.
- [ ] *측정 계획* 1건.
- [ ] *멘티* 1명.
- [ ] *결정 기록* 양식.

## 연습 문제

1. *systems thinking* 한 줄 정의.
2. *multiplier* *예* 한 줄.
3. *주니어 → 시니어* *가장 큰 변화* 한 줄.

## 정리 및 다음 단계

이 시리즈는 여기서 마무리됩니다. *데이터 커리어* 는 *기술 + 도메인 + 사람* 의 *합* 입니다.
