# 첫 직장 적응

> Developer Career 101 시리즈 (7/10)

<!-- a-grade-intro:begin -->

**핵심 질문**: *첫* *90일* 동안 *무엇* 을 *해야* *길게* *살아남나요*?

> *질문*, *기록*, *작은* *기여*.

<!-- a-grade-intro:end -->

## 이 글에서 배울 것

- *온보딩* 전략
- *질문* 의 *기술*
- *작은* *기여*
- *피드백* *루프*
- *관계* 만들기

## 왜 중요한가

*첫인상* 이 *18개월* 을 *결정* 합니다.

## 개념 한눈에 보기

```mermaid
flowchart LR
    L[Learn] --> A[Ask]
    A --> C[Contribute]
    C --> F[Feedback]
```

## 핵심 용어 정리

- **onboarding**: *적응* *과정*.
- **buddy**: *동료* *멘토*.
- **1:1**: *일대일* *대화*.
- **shadow**: *옆에서* *관찰*.
- **psychological safety**: *심리적* *안전감*.

## Before/After

**Before**: "*혼자* *공부* 만 *한다*."

**After**: "*매주* *질문* 을 *모으고* *PR* 을 *낸다*."

## 실습: 첫 90일 플랜

### 1단계 — 30일: 학습

```text
- 코드베이스 탐색
- 온콜 shadow
- 용어집 작성
```

### 2단계 — 60일: 질문 정리

```markdown
## 질문 노트
- 왜 X 가 Y 보다 빠른가?
- Z 의 retention 정책은?
```

### 3단계 — 90일: 첫 PR

```bash
git checkout -b first-fix
# small, scoped change
```

### 4단계 — 1:1 활용

```text
주 1회 매니저, 격주 mentor
```

### 5단계 — 회고

```markdown
- 잘한 것: 코드 리딩
- 부족한 것: 질문 빈도
- 다음: 매일 1개 질문
```

## 이 코드에서 주목할 점

- *질문* 이 *학습* 가속.
- *작은 PR* 이 *신뢰*.
- *1:1* 이 *피드백*.

## 자주 하는 실수 5가지

1. ***모르는* *것* 을 *숨긴다*.**
2. ***질문* 이 *너무* *크다*.**
3. ***PR* 이 *없다*.**
4. ***1:1* 을 *비운다*.**
5. ***기록* 이 *없다*.**

## 실무에서는 이렇게 쓰입니다

기업도 *90일* *목표* 와 *첫 PR* 을 *공식* *지표* 로 *삼는* 경우가 많습니다.

## 시니어 엔지니어는 이렇게 생각합니다

- *질문* 이 *예의*.
- *작은 PR* 이 *진도*.
- *1:1* 이 *신뢰*.
- *기록* 이 *경력*.
- *관찰* 이 *학습*.

## 체크리스트

- [ ] *용어집* 작성.
- [ ] *질문 노트*.
- [ ] *첫 PR*.
- [ ] *1:1* 정착.

## 연습 문제

1. *psychological safety* 한 줄 정의.
2. *buddy* 와 *mentor* 차이 한 줄.
3. *작은 PR* 의 *예* 한 줄.
