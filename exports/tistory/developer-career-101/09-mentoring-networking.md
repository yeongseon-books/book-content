
# 멘토링과 네트워킹

> Developer Career 101 시리즈 (9/10)

<!-- a-grade-intro:begin -->

**핵심 질문**: *멘토* 와 *네트워크* 는 *어떻게* *만드나요*?

> *작은* *기여*, *정중* 한 *요청*, *지속* *연락*.

<!-- a-grade-intro:end -->

## 이 글에서 배울 것

- *멘토 찾기*
- *질문 잘 하기*
- *커뮤니티* *참여*
- *컨퍼런스* *활용*
- *온라인* *프레즌스*

## 왜 중요한가

*혼자* *배우는* *속도* 는 *한계* 가 *있습니다*. *연결* 은 *지름길* 입니다.

## 개념 한눈에 보기

```mermaid
flowchart LR
    L[Learn] --> A[Ask]
    A --> G[Give back]
    G --> N[Network]
```

## 핵심 용어 정리

- **mentor**: *경험* *전수자*.
- **mentee**: *조언* *수신자*.
- **office hours**: *상담* *시간*.
- **community**: *공동체*.
- **paying it forward**: *되돌려* *주기*.

## Before/After

**Before**: "*혼자* *문서* 만 *읽는다*."

**After**: "*월* 1회 *멘토* *세션* + *주* 1회 *커뮤니티* *글*."

## 실습: 네트워크 만들기

### 1단계 — 멘토 후보 리스트

```text
- 직장 선배
- 오픈소스 메인테이너
- 블로그 저자
```

### 2단계 — 정중한 첫 메시지

```text
안녕하세요, X에 관심이 있어 Y를 시도 중입니다.
30분 시간 내주시면 Z를 묻고 싶습니다.
```

### 3단계 — 커뮤니티 참여

```text
- Discord/Slack 1개 선택
- 주 2회 답변 1개
```

### 4단계 — 컨퍼런스 후기

```bash
# 컨퍼런스 후 24시간 안에 후기 1편
```

### 5단계 — 온라인 프레즌스

```text
- GitHub README
- 블로그 1편/월
- LinkedIn 업데이트
```

## 이 코드에서 주목할 점

- *질문* 이 *준비* *되어야* *답* 이 *옵니다*.
- *기여* 가 *먼저* 가야 *연결* 이 *생깁니다*.
- *지속* 이 *신뢰* 입니다.

## 자주 하는 실수 5가지

1. ***대뜸* *멘토* *해달라*고 *한다*.**
2. ***질문* 이 *모호* *하다*.**
3. ***답례* 가 *없다*.**
4. ***감정* 만 *얹는다*.**
5. ***공개* *글* 이 *없다*.**

## 실무에서는 이렇게 쓰입니다

기업도 *멘토십* *프로그램* 과 *내부* *길드* 를 *운영* 합니다.

## 시니어 엔지니어는 이렇게 생각합니다

- *연결* 이 *복리*.
- *기여* 가 *먼저*.
- *질문* 은 *구체*.
- *되돌려* *주기*.
- *공개* 가 *기회*.

## 체크리스트

- [ ] *멘토* *후보* 3명.
- [ ] *커뮤니티* 1곳.
- [ ] *블로그* 1편/월.
- [ ] *답례* *습관*.

## 연습 문제

1. *mentor* 한 줄 정의.
2. *paying it forward* *예* 한 줄.
3. *좋은* *질문* 의 *기준* 한 줄.
