
# 시니어로 가는 길

> Developer Career 101 시리즈 (10/10)

<!-- a-grade-intro:begin -->

**핵심 질문**: *시니어* 는 *무엇* 이 *다른가요*?

> *범위*, *영향*, *판단*, *지속*.

<!-- a-grade-intro:end -->

## 이 글에서 배울 것

- *시니어* 의 *정의*
- *기술* *깊이*
- *영향력*
- *멘토링*
- *지속* *학습*

## 왜 중요한가

*시니어* 는 *직급* 이 *아니라* *행동* 입니다.

## 개념 한눈에 보기

```mermaid
flowchart LR
    J[Junior] --> M[Mid]
    M --> S[Senior]
    S --> L[Lead]
```

## 핵심 용어 정리

- **scope**: *책임* *범위*.
- **impact**: *영향*.
- **judgment**: *판단*.
- **ownership**: *소유*.
- **multiplier**: *증폭자*.

## Before/After

**Before**: "*티켓* 만 *처리* *한다*."

**After**: "*문제* 를 *정의* *하고* *해결* *방향* 을 *제안* *한다*."

## 실습: 시니어 행동 5가지

### 1단계 — 문제 정의

```text
"증상"이 아니라 "문제"를 적는다.
```

### 2단계 — 의사결정 기록

```markdown
- 결정
- 대안
- 근거
- 영향
```

### 3단계 — 멘토링 1명

```text
- 월 1회 1:1
- 의제는 멘티가 정한다
```

### 4단계 — 코드 리뷰 다섯 줄 룰

```text
- 1 칭찬
- 1 위험
- 1 대안
- 1 학습 포인트
- 1 다음 단계
```

### 5단계 — 학습 회고

```bash
# 분기마다 30분 회고를 적는다
```

## 이 코드에서 주목할 점

- *문서* 가 *영향력* 을 *증폭* 합니다.
- *멘토링* 이 *증폭자* 입니다.
- *회고* 가 *복리* 입니다.

## 자주 하는 실수 5가지

1. ***기술* 만 *판다*.**
2. ***결정* 을 *문서화* *안* *한다*.**
3. ***리뷰* 가 *공격적* *이다*.**
4. ***멘토링* 을 *미룬다*.**
5. ***회고* 를 *건너* *뛴다*.**

## 실무에서는 이렇게 쓰입니다

대기업의 *시니어* *경로* 는 *Staff*, *Principal* 로 *이어집니다*.

## 시니어 엔지니어는 이렇게 생각합니다

- *범위* 를 *넓힌다*.
- *영향* 을 *측정* *한다*.
- *판단* 을 *기록* *한다*.
- *동료* 를 *키운다*.
- *학습* 을 *멈추지* *않는다*.

## 체크리스트

- [ ] *문제 정의* 습관.
- [ ] *결정 문서* 양식.
- [ ] *멘토링* 1명.
- [ ] *분기 회고*.

## 연습 문제

1. *scope* 한 줄 정의.
2. *multiplier* *예* 한 줄.
3. *주니어 → 시니어* 의 *가장 큰 변화* 한 줄.
