
# 이력서와 포트폴리오

> Developer Career 101 시리즈 (4/10)

<!-- a-grade-intro:begin -->

**핵심 질문**: *채용 담당자* 가 *30초* 안에 *나* 를 *이해* 하게 *하려면* *어떻게* *써야* 할까요?

> *역할*, *결과*, *영향* 을 *수치* 로 *표현* 합니다.

<!-- a-grade-intro:end -->

## 이 글에서 배울 것

- *이력서* *구조*
- *불릿* 작성
- *포트폴리오* 와 *연결*
- *키워드* 와 *ATS*
- *지원 흐름*

## 왜 중요한가

*이력서* 는 *면접* 의 *입장권* 입니다.

## 개념 한눈에 보기

```mermaid
flowchart LR
    H[Header] --> S[Summary]
    S --> E[Experience]
    E --> P[Projects]
    P --> Sk[Skills]
```

## 핵심 용어 정리

- **summary**: *3줄* *요약*.
- **bullet**: *결과* *문장*.
- **STAR**: *Situation Task Action Result*.
- **ATS**: *지원자* *추적* *시스템*.
- **portfolio**: *증거* *모음*.

## Before/After

**Before**: "*담당 업무* 를 *나열* 한다."

**After**: "*결과* 와 *영향* 을 *수치* 로 *쓴다*."

## 실습: 이력서 작성

### 1단계 — Header

```text
이름 | 이메일 | GitHub | LinkedIn
```

### 2단계 — Summary

```text
3년차 백엔드. 결제 시스템 p95 200ms→80ms.
```

### 3단계 — Experience 불릿 (STAR)

```markdown
- Cut p95 latency from 200ms to 80ms by introducing read replicas, serving 5M req/day.
```

### 4단계 — Projects

```markdown
- tinytool: 1.2k stars, used by 30 orgs
```

### 5단계 — Skills

```text
Python, PostgreSQL, AWS, Kubernetes
```

## 이 코드에서 주목할 점

- *불릿* 은 *결과*.
- *수치* 가 *증거*.
- *STAR* 가 *서사*.

## 자주 하는 실수 5가지

1. ***담당 업무* 만 *적는다*.**
2. ***수치* 가 *없다*.**
3. ***포트폴리오* 와 *불일치* 한다.**
4. ***ATS* 가 *못 읽는* *형식* 을 *쓴다*.**
5. ***길이* 가 *3쪽* 을 *넘는다*.**

## 실무에서는 이렇게 쓰입니다

대기업도 *내부 이동* 시 *간단한* *내부 이력서* 를 *요구* 합니다.

## 시니어 엔지니어는 이렇게 생각합니다

- *이력서* 는 *광고*.
- *수치* 가 *왕*.
- *STAR* 는 *프레임*.
- *ATS* 는 *현실*.
- *포트폴리오* 가 *증거*.

## 체크리스트

- [ ] *Summary 3줄*.
- [ ] *불릿* 5개 *수치 포함*.
- [ ] *포트폴리오* 링크.
- [ ] *PDF + 텍스트* 두 *형식*.

## 연습 문제

1. *STAR* 한 줄 정의.
2. *ATS* 한 줄 정의.
3. *수치 불릿* *예* 한 줄.
