# 미분이란 무엇인가

미분은 변화율을 읽는 도구이고, 머신러닝 학습은 그 변화를 따라 손실을 줄이는 과정입니다.

이 글은 Calculus for ML 101 시리즈의 1번째 글입니다.

## 이 글에서 다룰 문제

- 머신러닝에서 말하는 "학습"은 왜 미분과 연결될까요?
- 평균 변화율과 순간 변화율은 무엇이 다를까요?
- 접선의 기울기와 도함수는 어떤 관계일까요?
- 극한은 왜 미분 정의에 꼭 필요할까요?
- 수치 미분은 언제 유용하고, 어디까지 믿어야 할까요?

> 미분은 함수가 한 점 근처에서 얼마나, 어느 방향으로 변하는지 알려 주는 도구입니다. 모델 학습은 이 변화를 읽어 손실을 줄이는 방향을 고르는 과정입니다.

> Calculus for ML 101 시리즈 (1/10)

## 이 글에서 배울 것

- 미분을 변화율 관점에서 이해합니다.
- 접선, 극한, 도함수의 연결을 봅니다.
- 수치 미분으로 직관을 확인합니다.
- 손실 함수의 기울기가 왜 학습 방향을 알려 주는지 이해합니다.

## 왜 중요한가

경사하강법, 역전파, 학습률은 모두 미분 위에 서 있습니다. 미분을 모르면 프레임워크가 내놓는 gradient 값을 숫자로만 보게 됩니다. 반대로 미분 감각이 있으면 지금 손실이 왜 줄어드는지, 왜 발산하는지, 왜 어느 방향으로 가중치를 움직여야 하는지 해석할 수 있습니다.

## 개념 한눈에 보기

## 핵심 용어

- **미분**: 한 점에서의 순간 변화율입니다.
- **기울기**: 두 점 사이에서 얼마나 올라가거나 내려가는지 나타내는 값입니다.
- **극한**: 어떤 값에 가까워질 때 함수가 접근하는 상태입니다.
- **접선**: 곡선의 한 점을 스치듯 지나가는 직선입니다.
- **수치 미분**: 정확한 식 대신 계산으로 근사한 미분값입니다.

## Before / After

**Before**: 손실이 왜 줄어드는지 감이 없습니다.

**After**: 손실의 기울기가 어느 방향으로 움직여야 하는지 알려 준다는 점을 이해합니다.

## 단계별 실습: 미니 미분 키트

### Step 1 — 함수 정의

```python
def f(x):
    return x ** 2
```

가장 단순한 제곱 함수를 준비합니다. 이 함수는 x가 커질수록 값이 빠르게 커지므로 변화율을 보기 좋습니다.

### Step 2 — 평균 변화율

```python
def avg_rate(f, a, b):
    return (f(b) - f(a)) / (b - a)
```

이 식은 구간 [a, b]에서의 평균 기울기입니다. 아직 미분은 아니지만, 미분으로 가기 전 첫 단계입니다.

### Step 3 — 수치 미분

```python
def deriv(f, x, h=1e-5):
    return (f(x + h) - f(x - h)) / (2 * h)
```

양옆에서 아주 조금 움직여 본 뒤 가운데 점의 기울기를 근사합니다. 중심 차분 방식이라 한쪽만 보는 것보다 안정적입니다.

### Step 4 — 접선의 기울기

```python
slope = deriv(f, 2.0)  # 약 4.0
```

x=2에서 기울기가 약 4라는 뜻입니다. 즉, 그 점 근처에서 x를 아주 조금 늘리면 함수값은 대략 4배 속도로 변합니다.

### Step 5 — 손실 직관

```python
def loss(w):
    return (w - 3) ** 2

g = deriv(loss, 0.0)   # 음수 -> w를 키워야 손실 감소
```

이제 미분을 손실 함수에 연결합니다. 기울기가 음수면 오른쪽으로 움직일수록 손실이 줄어든다는 뜻입니다.

## 이 코드에서 주목할 점

- 수치 미분은 정확한 해석 미분이 아니라 근사치입니다.
- 평균 변화율은 구간 전체를 보고, 미분은 한 점 근처를 봅니다.
- 도함수의 부호는 방향을, 크기는 변화의 강도를 알려 줍니다.

## 자주 하는 실수 5가지

1. h를 너무 작게 잡아 부동소수점 오차를 키웁니다.
2. 평균 변화율과 순간 변화율을 같은 것으로 봅니다.
3. 도함수 값과 원래 함수값을 혼동합니다.
4. 기울기의 크기만 보고 부호를 무시합니다.
5. 불연속점에서도 미분이 당연히 존재한다고 가정합니다.

## 실무에서는 이렇게 생각합니다

현업에서 수치 미분은 주로 디버깅과 검증에 씁니다. 실제 학습은 훨씬 빠른 해석 미분과 자동 미분으로 진행합니다. 그래도 수치 미분 직관을 알고 있으면 autograd가 계산한 gradient가 상식적으로 맞는지 확인할 수 있습니다.
