# 이력서와 포트폴리오

> Developer Career 101 시리즈 (4/10)


## 이 글에서 다룰 문제

*이력서* 는 *면접* 의 *입장권* 입니다.

## 전체 흐름
```mermaid
flowchart LR
    H[Header] --> S[Summary]
    S --> E[Experience]
    E --> P[Projects]
    P --> Sk[Skills]
```

## Before/After

**Before**: "*담당 업무* 를 *나열* 한다."

**After**: "*결과* 와 *영향* 을 *수치* 로 *쓴다*."

## 이력서 작성

### 1단계 — Header

```text
이름 | 이메일 | GitHub | LinkedIn
```

### 2단계 — Summary

```text
3년차 백엔드. 결제 시스템 p95 200ms→80ms.
```

### 3단계 — Experience 불릿 (STAR)

```markdown
- Cut p95 latency from 200ms to 80ms by introducing read replicas, serving 5M req/day.
```

### 4단계 — Projects

```markdown
- tinytool: 1.2k stars, used by 30 orgs
```

### 5단계 — Skills

```text
Python, PostgreSQL, AWS, Kubernetes
```

## 이 코드에서 주목할 점

- *불릿* 은 *결과*.
- *수치* 가 *증거*.
- *STAR* 가 *서사*.

## 자주 하는 실수 5가지

1. ***담당 업무* 만 *적는다*.**
2. ***수치* 가 *없다*.**
3. ***포트폴리오* 와 *불일치* 한다.**
4. ***ATS* 가 *못 읽는* *형식* 을 *쓴다*.**
5. ***길이* 가 *3쪽* 을 *넘는다*.**

## 실무에서는 이렇게 쓰입니다

대기업도 *내부 이동* 시 *간단한* *내부 이력서* 를 *요구* 합니다.

## 체크리스트

- [ ] *Summary 3줄*.
- [ ] *불릿* 5개 *수치 포함*.
- [ ] *포트폴리오* 링크.
- [ ] *PDF + 텍스트* 두 *형식*.
