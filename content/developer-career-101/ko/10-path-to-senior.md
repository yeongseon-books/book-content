# 시니어로 가는 길

> Developer Career 101 시리즈 (10/10)


## 이 글에서 다룰 문제

*시니어* 는 *직급* 이 *아니라* *행동* 입니다.

## 전체 흐름
```mermaid
flowchart LR
    J[Junior] --> M[Mid]
    M --> S[Senior]
    S --> L[Lead]
```

## Before/After

**Before**: "*티켓* 만 *처리* *한다*."

**After**: "*문제* 를 *정의* *하고* *해결* *방향* 을 *제안* *한다*."

## 시니어 행동 5가지

### 1단계 — 문제 정의

```text
"증상"이 아니라 "문제"를 적는다.
```

### 2단계 — 의사결정 기록

```markdown
- 결정
- 대안
- 근거
- 영향
```

### 3단계 — 멘토링 1명

```text
- 월 1회 1:1
- 의제는 멘티가 정한다
```

### 4단계 — 코드 리뷰 다섯 줄 룰

```text
- 1 칭찬
- 1 위험
- 1 대안
- 1 학습 포인트
- 1 다음 단계
```

### 5단계 — 학습 회고

```bash
# 분기마다 30분 회고를 적는다
```

## 이 코드에서 주목할 점

- *문서* 가 *영향력* 을 *증폭* 합니다.
- *멘토링* 이 *증폭자* 입니다.
- *회고* 가 *복리* 입니다.

## 자주 하는 실수 5가지

1. ***기술* 만 *판다*.**
2. ***결정* 을 *문서화* *안* *한다*.**
3. ***리뷰* 가 *공격적* *이다*.**
4. ***멘토링* 을 *미룬다*.**
5. ***회고* 를 *건너* *뛴다*.**

## 실무에서는 이렇게 쓰입니다

대기업의 *시니어* *경로* 는 *Staff*, *Principal* 로 *이어집니다*.

## 체크리스트

- [ ] *문제 정의* 습관.
- [ ] *결정 문서* 양식.
- [ ] *멘토링* 1명.
- [ ] *분기 회고*.
