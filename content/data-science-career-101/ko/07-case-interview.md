# 케이스 인터뷰

> Data Science Career 101 시리즈 (7/10)


## 이 글에서 다룰 문제

*기술* 만 *있고* *제품 감각* 이 *없으면* *떨어집니다*.

## 개념 한눈에 보기

```mermaid
flowchart LR
    Q[Clarify] --> M[Metric]
    M --> H[Hypothesis]
    H --> D[Data]
    D --> Dec[Decision]
```

## Before/After

**Before**: "*숫자* 만 *답* *한다*."

**After**: "*문제 → 지표 → 가설 → 데이터 → 결정* 으로 *답* *한다*."

## 실습: 5단계 프레임

### 1단계 — 명확화

```text
"DAU가 떨어졌다 = 어느 세그먼트, 어느 기간?"
```

### 2단계 — 지표

```text
- 북극성 지표
- 보조 지표 2개
```

### 3단계 — 가설

```text
- 기능 변화
- 외부 이벤트
- 데이터 파이프라인
```

### 4단계 — 데이터

```text
- 어떤 쿼리, 어떤 비교
- A/B 가능 여부
```

### 5단계 — 결정

```text
- 추천 액션
- 리스크
- 후속 측정
```

## 이 코드에서 주목할 점

- *명확화* 가 *시간* 을 *법니다*.
- *지표* 가 *대화* 의 *축*.
- *결정* 까지 *내려야* *완성*.

## 자주 하는 실수 5가지

1. ***즉답* 한다.**
2. ***지표* 가 *없다*.**
3. ***가설* 이 *하나*.**
4. ***데이터* *제안* 이 *모호*.**
5. ***결정* 없이 *끝* *난다*.**

## 실무에서는 이렇게 쓰입니다

빅테크 *PM/사이언티스트 인터뷰* 의 50% 이상이 *케이스* 입니다.

## 체크리스트

- [ ] *명확화 질문* 3개.
- [ ] *지표* 1개+보조.
- [ ] *가설* 3개.
- [ ] *결정* 1개.
