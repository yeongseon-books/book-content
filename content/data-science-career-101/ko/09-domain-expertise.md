# 도메인 전문성 쌓기

데이터 직무에서 기술은 분명 중요합니다. 하지만 시간이 지날수록 더 크게 차이를 만드는 것은 도메인 이해입니다. 같은 SQL을 써도 전자상거래, 핀테크, 게임, 헬스케어는 중요한 지표와 질문이 전혀 다르기 때문입니다. 기술은 어느 정도 복제 가능하지만, 도메인 감각은 경험이 쌓일수록 더 큰 격차를 만듭니다.

그래서 좋은 데이터 실무자는 대시보드를 잘 만드는 사람에서 그치지 않습니다. 업계 용어를 알고, 핵심 KPI를 이해하고, 현장의 운영 흐름을 읽고, 외부 변화가 숫자에 어떤 영향을 주는지 연결할 수 있어야 합니다. 도메인 전문성은 기술 위에 올라가는 추가 옵션이 아니라, 해석의 정확도를 끌어올리는 기반입니다.

## 이 글에서 다룰 문제

- 데이터 직무에서 도메인 전문성은 왜 중요할까요?
- 도메인 용어와 KPI는 어떻게 익히는 편이 좋을까요?
- 현장 관찰이 숫자 해석 능력을 높이는 이유는 무엇일까요?
- 외부 학습을 병행해야 하는 이유는 무엇일까요?
- 도메인 이해를 일회성이 아니라 루프로 만들어야 하는 이유는 무엇일까요?

> 기술이 데이터를 다루게 해 준다면, 도메인은 그 데이터가 실제로 무엇을 뜻하는지 읽게 해 줍니다.

## 한눈에 보는 전체 흐름

```mermaid
flowchart LR
    V[용어] --> M[지표]
    M --> F[현장]
    F --> L[반복 루프]
```

도메인 학습은 보통 이 순서로 쌓입니다. 먼저 용어를 알아야 회의가 들리고, 지표를 알아야 숫자가 읽히고, 현장을 알아야 왜 그런 숫자가 나오는지 감이 붙습니다. 이후에는 외부 학습과 회고를 통해 계속 업데이트해야 합니다.

## 핵심 용어

- **domain**: 특정 산업이나 비즈니스 영역입니다.
- **glossary**: 용어와 정의를 정리한 목록입니다.
- **KPI**: 핵심 성과 지표입니다.
- **playbook**: 반복 상황에서 따르는 운영 가이드입니다.
- **shadowing**: 현장 업무를 옆에서 관찰하며 배우는 방식입니다.

## Before/After

**Before**: "업계를 잘 모르는 상태에서 대시보드 숫자만 바꿔 왔다."

**After**: "KPI와 업계 용어를 기준으로 비즈니스 대화를 이어 갈 수 있다."

## 실습: 도메인 학습 다섯 단계

### Step 1 — Build a Glossary

```text
- 용어 30개
- 정의 + 예시
```

도메인 학습의 시작은 어휘입니다. 회의에서 자주 나오는 단어 30개만 정리해도 이해도가 크게 달라집니다. 단순 정의만 적지 말고, 실제 예시까지 붙여 두면 훨씬 오래 남습니다.

### Step 2 — Five Key Metrics

```text
- 정의
- 수식
- 사용하는 팀
```

지표는 이름만 알면 부족합니다. 계산식이 무엇인지, 어느 팀이 주로 쓰는지, 어떤 의사결정에 연결되는지까지 같이 알아야 해석이 깊어집니다.

### Step 3 — Field Shadowing

```text
- 영업 또는 운영과 하루 동행
- 메모와 질문
```

현장을 보는 경험은 숫자 뒤의 맥락을 이해하는 데 매우 중요합니다. 예를 들어 운영팀의 실제 업무 흐름을 한 번만 봐도, 같은 지표를 왜 그렇게 정의하는지 금방 납득되는 경우가 많습니다.

### Step 4 — External Study

```text
- 분기마다 업계 컨퍼런스 하나
- 업계 뉴스 RSS
```

내부 데이터만 보면 조직 안 시야에 갇히기 쉽습니다. 분기마다 업계 컨퍼런스를 보거나 뉴스 흐름을 따라가면, 외부 변화가 내부 지표에 어떻게 반영되는지도 더 잘 읽힙니다.

### Step 5 — Quarterly Retro

```text
- 새 용어 열 개
- 새 지표 세 개
```

도메인 전문성도 회고가 필요합니다. 새로 익힌 단어와 지표를 분기마다 점검하면, 지식이 흩어지지 않고 축적됩니다.

## 이 예시에서 봐야 할 점

- 어휘가 도메인 이해의 입구입니다.
- 지표가 방향을 잡아 줍니다.
- 현장을 봐야 숫자의 진짜 의미가 보입니다.

입문자는 기술 공부만으로도 바쁘기 때문에 도메인 학습을 뒤로 미루기 쉽습니다. 하지만 실제 현업에서는 도메인을 모르면 좋은 질문을 만들기 어렵고, 같은 그래프를 봐도 중요한 변화인지 아닌지 판단하기 어렵습니다. 결국 도메인 이해는 분석 품질을 끌어올리는 지름길입니다.

## 자주 하는 실수 5가지

1. **기술만 깊게 파고 도메인을 미루는 실수**
2. **용어집 없이 회의를 버티려는 실수**
3. **지표 정의를 모호하게 아는 실수**
4. **현장 운영을 전혀 보지 않는 실수**
5. **외부 학습을 건너뛰는 실수**

## 실무에서는 이렇게 나타납니다

핀테크, 헬스케어, 게임처럼 규제나 사용자 행동 패턴이 강한 산업일수록 도메인이 답을 바꿉니다. 같은 리텐션 지표라도 업종마다 정상 범위가 다르고, 같은 매출 감소라도 계절성이나 외부 정책 변화 때문에 해석이 달라질 수 있습니다. 그래서 도메인을 아는 데이터 실무자는 훨씬 빠르게 본질에 접근합니다.

## 시니어는 이렇게 생각합니다

- 용어부터 정확히 익힙니다.
- 지표 정의를 흐리지 않습니다.
- 현장을 직접 관찰하려고 합니다.
- 외부 변화도 같이 공부합니다.
- 도메인 학습을 일회성이 아니라 반복 루프로 만듭니다.

## 체크리스트

- [ ] 핵심 용어 30개를 정리했다.
- [ ] 주요 KPI 다섯 개를 정의와 함께 적어 봤다.
- [ ] 현장 동행 또는 shadowing 일정을 잡았다.
- [ ] 분기 회고 방식까지 정했다.

## 연습 문제

1. KPI를 한 줄로 설명해 보세요.
2. playbook의 예를 한 줄로 적어 보세요.
3. 좋은 도메인 학습의 기준을 한 줄로 정리해 보세요.

## 정리 및 다음 단계

데이터 직무에서 기술은 출발점이지만, 도메인 전문성은 해석의 질을 바꾸는 힘입니다. 용어집을 만들고, 핵심 KPI를 정리하고, 현장을 관찰하고, 외부 학습을 병행하면 숫자 뒤의 맥락이 훨씬 선명해집니다. 결국 좋은 데이터 실무자는 데이터를 많이 아는 사람이라기보다, 그 데이터가 속한 세계를 더 잘 이해하는 사람에 가깝습니다.

다음 글에서는 이런 경험과 도메인 이해를 바탕으로 시니어 데이터 직무로 성장할 때 무엇이 달라지는지 살펴보겠습니다.
