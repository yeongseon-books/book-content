# 머신러닝에서의 확률

확률 시리즈를 끝까지 읽고 나면 자연스럽게 이런 질문이 남습니다. “좋다. 표본공간, 조건부확률, 분포, 기대값까지는 알겠다. 그런데 이게 실제 머신러닝 안에서는 어디에 들어가 있나?” 이 질문에 답하지 못하면 확률은 교양으로 남고, 답할 수 있으면 확률은 도구가 됩니다.

사실 현대 머신러닝의 많은 부분은 확률을 다른 이름으로 부르고 있을 뿐입니다. 분류 모델의 출력 확률, 교차 엔트로피 손실, 음의 로그가능도, 확률 보정, 베이지안 추론, 불확실성 추정이 모두 같은 뿌리에서 나옵니다. 모델이 예측을 잘하는지 묻는 순간에도, 모델이 얼마나 확신하는지 묻는 순간에도 확률이 등장합니다.

이 글에서는 손실함수와 가능도, 분류기 출력의 의미, 확률 보정, 베이지안 업데이트를 한 흐름으로 묶어 보겠습니다. 시리즈 마지막 글답게, 지금까지 배운 확률 개념이 머신러닝 시스템 안에서 어디에 자리 잡는지 연결하는 데 초점을 두겠습니다.

## 이 글에서 다룰 문제

- 머신러닝에서 확률은 정확히 어느 지점에 숨어 있을까요?
- 교차 엔트로피와 음의 로그가능도는 왜 사실상 같은 이야기를 할까요?
- 분류 모델이 내놓는 0.8은 무엇을 의미하고, 언제 믿으면 안 될까요?
- 확률 보정은 정확도와 어떻게 다른 축의 품질을 말할까요?
- 베이지안 관점은 ML에서 언제 특히 유용할까요?

> 머신러닝은 정답을 찍는 기계가 아니라, 관측된 데이터를 바탕으로 불확실성을 정리하는 기계에 가깝습니다.

## 왜 중요한가

모델이 0 또는 1만 내놓는다면 해석은 단순합니다. 하지만 실제 모델은 대개 점수를 냅니다. 스팸일 확률, 구매할 확률, 질환이 있을 확률, 장애가 날 확률처럼 예측은 거의 언제나 확률의 형태를 띱니다. 이 숫자를 이해하지 못하면 모델이 틀렸는지, 과신하는지, 임계값을 잘못 잡았는지 분리해서 볼 수 없습니다.

또 하나 중요한 이유는 학습 자체가 확률 최적화와 맞닿아 있기 때문입니다. 분류에서는 교차 엔트로피를 최소화하고, 회귀에서는 특정 오차 분포를 가정한 가능도 최대화 문제로 볼 수 있습니다. 즉 손실함수는 단순한 계산 편의 장치가 아니라 “이 데이터가 어떤 확률 모형에서 나왔다고 보는가”를 반영합니다.

그래서 머신러닝에서 확률을 안다는 말은 공식을 외운다는 뜻이 아닙니다. 모델이 내놓은 숫자를 어떤 행동으로 바꿀지, 그 숫자가 과신인지 과소신인지, 운영 환경이 달라졌을 때 그 해석이 유지되는지까지 함께 본다는 뜻입니다. 예측 모델을 만드는 일과 예측을 믿어도 되는지 판단하는 일은 결국 하나의 흐름입니다.

## 핵심 개념 한눈에 보기

```mermaid

    Data["데이터"] --> Model["모델 p(y|x; theta)"]

    Train --> Inf["추론: p(y|x)"]
    Inf --> Eval["Calibration / AUC"]
```

## 핵심 용어

- **가능도**: 주어진 파라미터 θ 아래에서 데이터가 얼마나 그럴듯한지를 보는 함수입니다.
- **MLE**: 가능도를 가장 크게 만드는 θ를 찾는 방식입니다.
- **MAP**: 사전분포와 가능도를 함께 고려해 사후확률을 가장 크게 만드는 방식입니다.
- **교차 엔트로피**: 예측분포와 실제 레이블 사이의 차이를 재는 대표 손실입니다.
- **확률 보정**: 예측 확률이 실제 빈도와 얼마나 잘 맞는지 보는 성질입니다.

이 다섯 용어는 서로 따로 떨어져 있지 않습니다. 가능도에 로그를 취하고 부호를 뒤집으면 최적화하기 쉬운 손실이 되고, 그 손실을 줄여 얻은 모델이 내놓는 출력은 다시 확률로 해석됩니다. 그 확률이 실제 빈도와 맞는지 보는 단계가 확률 보정입니다.

즉 학습, 추론, 평가는 각각 따로 떨어진 단계가 아닙니다. 학습에서 어떤 손실을 택했는지가 추론 단계의 출력 의미를 정하고, 그 출력이 실제 세계와 얼마나 맞는지가 평가 단계에서 드러납니다. 확률을 이해하면 세 단계를 하나의 언어로 읽을 수 있습니다.

## Before / After

**Before**: “모델 출력이 0.8이다.”라고 들었지만, 이 값이 점수인지 확률인지, 임계값 기준인지, 신뢰도인지 구분이 흐립니다.

**After**: “이 값은 p(y=1 | x)=0.8에 대한 추정치다. 다만 확률 보정이 나쁘면 0.8이라는 숫자를 곧장 80%의 실제 빈도로 믿으면 안 된다.”라고 설명할 수 있습니다.

이 차이가 중요합니다. 숫자를 쓰는 것과 숫자를 해석하는 것은 다른 역량입니다. 머신러닝 운영에서 문제가 되는 쪽은 대개 후자입니다.

## 5단계로 보는 ML 확률 직관

### Step 1 — 교차 엔트로피 손실

```python
import numpy as np
y = np.array([1, 0, 1, 1, 0])
p = np.array([0.9, 0.2, 0.8, 0.6, 0.3])
nll = -np.mean(y*np.log(p) + (1-y)*np.log(1-p))
print("NLL:", nll)
```

이 식은 이진 분류에서 매우 자주 보는 손실입니다. 이름은 교차 엔트로피이지만, 확률 관점에서는 음의 로그가능도로 읽을 수 있습니다. 정답 레이블에 높은 확률을 줄수록 손실이 작아지고, 틀린 확신을 크게 줄수록 손실이 커집니다. 그래서 이 손실은 단순히 맞고 틀림만 세는 방식보다 훨씬 많은 정보를 담습니다.

### Step 2 — 로지스틱 회귀 출력 읽기

```python
import numpy as np
from sklearn.linear_model import LogisticRegression
X = np.array([[0],[1],[2],[3],[4]])
y = np.array([0, 0, 1, 1, 1])
clf = LogisticRegression().fit(X, y)
print("p(y=1|x=2):", clf.predict_proba([[2]])[0, 1])
```

`predict_proba`가 내놓는 값은 단순 점수가 아니라 조건부확률의 추정치로 읽습니다. 정확히는 모델이 학습한 가정 아래에서 입력 x가 주어졌을 때 y=1일 확률입니다. 이 해석이 가능한 이유는 로지스틱 회귀가 본질적으로 확률 모형이기 때문입니다.

### Step 3 — 확률 보정 확인하기

```python
import numpy as np
# 예측확률 vs 실제 비율
preds = np.array([0.1, 0.3, 0.5, 0.7, 0.9])
actual = np.array([0.12, 0.28, 0.55, 0.66, 0.91])
print("calibration gap:", np.abs(preds - actual).mean())
```

정확도가 높다고 확률 보정이 좋은 것은 아닙니다. 어떤 모델은 정답은 잘 맞히지만 확률을 지나치게 자신 있게 내놓습니다. 예를 들어 0.9라고 예측한 표본들이 실제로는 70%만 정답이라면, 그 모델은 과신하고 있는 것입니다. 확률을 내놓는 모델이라면 이 차이를 반드시 봐야 합니다.

### Step 4 — 베이지안 업데이트 (개념)

```python
# 사전 p(theta)와 가능도 p(D|theta)에서 사후 도출
prior = 0.5
likelihood = 0.8
post = likelihood * prior / (likelihood * prior + (1 - likelihood) * (1 - prior))
print("posterior:", post)
```

베이지안 관점은 모델 파라미터나 가설에 대한 믿음을 데이터로 갱신하는 방식입니다. 딥러닝 실무에서 모든 파라미터에 대해 완전한 베이지안 추론을 하지는 않더라도, prior를 둔다거나 MAP 추정을 쓴다거나, 불확실성을 모델링한다는 사고방식은 매우 자주 쓰입니다.

### Step 5 — 브라이어 점수

```python
import numpy as np
y = np.array([1, 0, 1, 0])
p = np.array([0.9, 0.2, 0.6, 0.4])
brier = np.mean((p - y)**2)
print("Brier:", brier)
```

브라이어 점수는 예측확률과 실제 결과 사이의 제곱오차를 봅니다. 로그 손실과 달리 해석이 비교적 직관적이고, 확률 보정 관점에서도 자주 언급됩니다. 어떤 지표를 쓰든 핵심은 같습니다. 모델이 “맞혔는가”뿐 아니라 “얼마나 적절한 확신으로 맞혔는가”를 보아야 합니다.

## 이 코드에서 주목할 점

- 교차 엔트로피는 확률 관점에서 음의 로그가능도와 이어집니다.
- 로지스틱 회귀의 출력은 조건부확률 p(y|x)로 읽을 수 있습니다.
- 확률 보정은 정확도와 별개의 평가축입니다.
- 베이지안 사고방식은 사전 정보와 데이터 업데이트를 함께 다룹니다.
- 브라이어 점수와 로그 손실은 확률 예측의 품질을 보는 대표 지표입니다.

## 자주 하는 실수 5가지

1. **원시 점수를 곧바로 확률로 읽는 실수**

   모든 모델 출력이 확률은 아닙니다. margin, logit, score를 그대로 확률처럼 다루면 해석이 무너집니다.

2. **확률 보정을 보지 않는 실수**

   정확도가 좋아도 확률이 틀릴 수 있습니다. 의사결정 비용이 큰 문제일수록 확률 보정을 따로 봐야 합니다.

3. **accuracy만으로 평가를 끝내는 실수**

   클래스 불균형이 있거나 확률 출력이 중요한 문제에서는 accuracy 하나로는 모델의 품질을 설명할 수 없습니다.

4. **불균형 데이터에 임계값 0.5를 기계적으로 쓰는 실수**

   90:10 데이터에서 0.5 임계값은 비용 구조를 전혀 반영하지 못할 수 있습니다. threshold는 확률과 비용을 함께 보고 정해야 합니다.

5. **사전 정보가 없다고 가정하는 실수**

   prior를 명시하지 않을 뿐, 실제로는 모델 구조, 정규화, 데이터 수집 방식이 이미 강한 사전 정보를 넣고 있는 경우가 많습니다.

## 실무에서는 이렇게 드러납니다

스팸 분류, 의료 진단, 사기 탐지, 추천 시스템, 이상치 탐지 같은 문제에서는 모델 점수를 어떻게 행동으로 바꿀지가 핵심입니다. 이때 필요한 것이 확률 해석입니다. 예측확률이 높아도 오탐 비용이 크면 임계값을 올려야 하고, 반대로 놓치면 큰 손실이 나는 문제라면 낮춰야 합니다.

또한 운영 환경에서는 확률 보정의 드리프트도 봐야 합니다. 학습 당시에는 0.8이 실제 80%와 비슷했더라도, 데이터 분포가 바뀌면 이 대응이 깨질 수 있습니다. 확률을 내놓는 모델은 학습으로 끝나지 않고, 시간이 지나도 그 확률이 여전히 정직한지 계속 점검해야 합니다.

특히 사람의 판단이 뒤에 이어지는 시스템일수록 이 차이는 더 큽니다. 검토 대상을 줄이기 위한 알림 시스템, 자동 심사 보조 도구, 의료 보조 모델처럼 확률이 곧 우선순위가 되는 문제에서는 숫자 하나의 의미가 곧 업무 흐름을 바꿉니다. 이런 환경에서는 잘 맞히는 모델보다 정직하게 확신을 표현하는 모델이 더 안전한 경우도 많습니다.

## 숙련자는 이렇게 생각합니다

- 손실함수를 볼 때 수학 공식만 보지 않고 어떤 확률 가정을 깔고 있는지 함께 봅니다.
- 정확도와 확률 보정을 분리해서 평가합니다.
- 임계값은 0.5로 고정하지 않고 비용과 목적에 맞게 정합니다.
- 불확실성이 중요한 문제에서는 예측값 하나보다 분포나 구간을 더 중시합니다.
- 빈도주의 도구와 베이지안 도구를 상황에 맞게 함께 사용합니다.

## 마무리

머신러닝에서 확률은 주변 개념이 아니라 중심 개념입니다. 손실함수 안에도, 예측 출력 안에도, 평가 지표 안에도, 불확실성 해석 안에도 확률이 들어 있습니다. 이 글에서 꼭 남겨야 할 핵심은 세 가지입니다. 손실은 종종 확률 모형의 다른 표현이라는 점, 모델 출력은 확률 보정을 거쳐야 비로소 신뢰할 만한 확률이 된다는 점, 그리고 베이지안 사고방식은 머신러닝에서 사전 정보와 데이터 업데이트를 연결하는 실용적인 렌즈라는 점입니다.

이 글로 Probability 101 시리즈를 마칩니다. 다음 단계는 선형대수와 머신러닝 자체의 모델 구조를 배우는 것입니다. 확률이 모델의 불확실성을 다뤘다면, 그다음 공부는 모델이 입력을 어떻게 표현하고 변환하는지를 다루게 됩니다.

시리즈 전체를 관통하는 한 문장을 남기면 이렇습니다. 머신러닝은 숫자를 뽑아내는 기술이 아니라, 불확실한 세계에서 더 나은 결정을 돕는 기술입니다. 확률을 이해하면 모델의 자신감과 한계를 함께 읽을 수 있고, 그 감각이 결국 더 안전한 시스템으로 이어집니다.
