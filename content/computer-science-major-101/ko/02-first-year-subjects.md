# Computer Science Major 101 (2/10): 1학년 과목 이해하기

이 글은 Computer Science Major 101 시리즈의 2번째 글입니다.

1학년 시간표를 보면 전공다운 이름보다 수학과 기초 과목이 먼저 눈에 들어옵니다. 코딩을 배우러 왔다고 생각한 학생에게는 이 구성이 다소 멀게 느껴질 수 있습니다.

![Computer Science Major 101 2장 흐름 개요](https://yeongseon-books.github.io/book-public-assets/assets/computer-science-major-101/02/02-01-first-year-foundation-map.ko.png)
*컴퓨터학과 전공 가이드 2장 흐름 개요*
> 1학년의 핵심은 각 과목을 따로 결들 기본씨 난중 비중 단순히 돌링음이 되는 것이 아니라, 각 것이 다음 게 다식 눈을 되넘수 있는 부분의 네이라는 데 있습니다.

## 먼저 던지는 질문

- 왜 1학년 과목은 수학과 프로그래밍 비중이 유난히 클까요?
- 미적분, 선형대수, 이산수학, 프로그래밍 입문은 각각 어떤 역할을 맡을까요?
- 지금은 진로와 멀어 보이는 과목이 왜 2학년 이후 성과를 가를까요?

## 이 글에서 배울 것

- 미적분의 의미
- 선형대수의 역할
- 이산수학의 쓰임
- 프로그래밍 입문 과목의 성격
- 공부 습관의 중요성

## 왜 중요한가

1학년 기초는 알고리즘, AI, 시스템처럼 뒤에 나오는 모든 과목의 바닥입니다. 이 시기에 쌓은 기초가 약하면 상위 과목에서 계속 같은 구멍을 다시 메우게 됩니다.

## 한눈에 보는 개념

> 1학년 과목은 나중에 다시 만날 개념을 미리 깔아 두는 기반 공사에 가깝습니다.

과목은 병렬로 들을 수 있어도 이해의 흐름은 서로 연결됩니다. 변화량을 다루는 힘, 벡터와 행렬 감각, 논리와 집합 사고, 기본 코딩 습관이 함께 쌓여야 다음 학년 과목이 버텨집니다.

## 핵심 용어

- **미적분(calculus)**: 연속적으로 변하는 값을 다루는 수학입니다.
- **선형대수(linear algebra)**: 벡터와 행렬을 중심으로 구조를 보는 수학입니다.
- **이산수학(discrete math)**: 논리와 집합, 그래프를 다루는 수학입니다.
- **프로그래밍 입문(intro programming)**: 언어와 프로그램 기본기를 배우는 과목입니다.
- **실습(lab)**: 손으로 직접 구현해 보는 시간입니다.

## 바뀌는 지점

**Before**: 1학년 과목이 진로와 멀어 보입니다.

**After**: 이후 모든 전공 과목의 뿌리라는 점이 보입니다.

## 실습: 1학년 과목 매트릭스

### 1단계 — 과목 나열

```python
courses = ["calculus", "linalg", "discrete", "intro_prog"]
```

학교마다 과목명은 조금씩 달라도 1학년 핵심 축은 대체로 이 범위에서 크게 벗어나지 않습니다.

### 2단계 — 상위 과목 연결

```python
maps = {"calculus": "ml", "linalg": "ml", "discrete": "algorithms", "intro_prog": "all"}
```

각 과목이 어디로 이어지는지 적어 보면 목적이 분명해집니다. 특히 프로그래밍 입문은 사실상 모든 후속 과목의 공통 기반입니다.

### 3단계 — 주당 학습 시간

```python
hours = {c: 6 for c in courses}
```

기초 과목은 감으로 공부하면 쉽게 밀립니다. 주당 시간을 먼저 배정해 두면 체감 난이도에 따라 한 과목만 붙잡는 실수를 줄일 수 있습니다.

### 4단계 — 실습 비중

```python
labs = {"intro_prog": 4, "discrete": 1}
```

프로그래밍은 강의를 이해하는 것과 직접 손을 움직이는 것이 별개입니다. 실습 시간을 따로 확보하지 않으면 문법은 아는데 구현이 안 되는 상태가 오래 갑니다.

### 5단계 — 약점 표시

```python
weak = [c for c, h in hours.items() if h < 5]
```

약한 과목을 빨리 찾는 편이 유리합니다. 초반 약점은 시간이 갈수록 더 비싼 복습 비용으로 돌아오기 때문입니다.

## 이 코드에서 먼저 볼 점

- 각 과목은 상위 과목과 연결됩니다.
- 학습 시간은 곧 기초 체력입니다.
- 실습 시간은 별도로 계산해야 합니다.

## 자주 하는 실수 5가지

1. **출석만 하고 복습하지 않는 일입니다.**
2. **수학을 이해가 아니라 암기로만 버티는 일입니다.**
3. **언어를 자꾸 바꾸면서 기초를 미루는 일입니다.**
4. **실습 과제를 마감 직전에 시작하는 일입니다.**
5. **질문하지 않고 혼자 오래 끌고 가는 일입니다.**

## 실무에서는 이렇게 드러납니다

실무에서 바로 미적분 문제를 풀지는 않더라도, 논리와 구조가 흔들리면 코드 리뷰와 문제 해결 방식도 함께 흔들립니다. 상위 과목과 현업 역량은 생각보다 1학년 기초 위에 정직하게 서 있습니다.

## 선배 엔지니어는 이렇게 봅니다

- 기초는 일찍 쌓을수록 복리처럼 남습니다.
- 수학은 짧게 외우기보다 천천히 이해하는 편이 오래 갑니다.
- 코드는 매일 조금씩 보는 습관이 중요합니다.
- 실수는 실패가 아니라 학습 기록입니다.
- 질문은 초반에 붙일수록 강한 습관이 됩니다.

## 1학년 과목을 전략적으로 듣는 프레임

1학년은 가장 화려한 주제를 배우는 시기가 아니라, 나중에 화려한 주제를 버틸 수 있게 만드는 시기입니다. 그래서 과목 선택 기준도 흥미 우선이 아니라 누적 효율 우선이어야 합니다. 특히 미적분, 선형대수, 이산수학, 프로그래밍 입문은 서로 다른 과목처럼 보이지만 실제로는 같은 문제 해결 파이프라인의 각 단계입니다. 미적분은 변화량과 최적화 직관을 만들고, 선형대수는 다차원 구조를 다루는 문법을 주고, 이산수학은 논리와 증명 습관을 고정하고, 프로그래밍 입문은 이를 실행 가능한 절차로 바꿉니다.

아래 표는 첫 언어 선택과 수학 과목 학습을 함께 묶어 판단할 때 유용합니다.

| 판단 항목 | Python 중심 시작 | C 중심 시작 | Java 중심 시작 |
| --- | --- | --- | --- |
| 초반 진입 장벽 | 낮음 | 중간~높음 | 중간 |
| 문법 대비 개념 학습 속도 | 빠름 | 중간 | 중간 |
| 메모리/저수준 감각 | 상대적으로 약함 | 강함 | 중간 |
| 자료구조 과목 연결 | 좋음 | 매우 좋음 | 좋음 |
| 웹·데이터 실습 연결 | 매우 좋음 | 보통 | 좋음 |
| 초심자 좌절 위험 | 낮음 | 높음 | 중간 |

중요한 점은 어느 언어가 절대적으로 우월하다는 결론이 아니라, "첫 언어의 목적"을 분명히 하는 것입니다. 1학년에서 필요한 것은 멀티언어 경험보다 한 언어로 함수 분해, 조건 설계, 예외 처리, 테스트 습관을 고정하는 일입니다. 언어를 자주 바꾸면 문법은 늘지만 사고 체계가 약해질 수 있습니다.

## 수학 과목과 코딩 과목을 분리하지 않는 학습법

많은 학생이 수학은 시험용, 코딩은 프로젝트용으로 분리해 생각합니다. 하지만 실제 전공 성과는 두 영역의 연결에서 결정됩니다. 예를 들어 이산수학의 명제 논리를 이해하면 조건문 설계에서 경계 사례를 더 정확히 다루게 됩니다. 선형대수의 벡터 감각은 머신러닝뿐 아니라 추천 점수 계산, 임베딩 유사도 해석에도 직접 연결됩니다. 미적분의 변화율 개념은 최적화 과정에서 학습률과 손실 감소를 읽는 기본 직관이 됩니다.

아래는 1학년 한 학기 운영에 바로 적용할 수 있는 최소 루프입니다.

- 강의 직후 24시간 안에 핵심 개념 3줄 요약
- 48시간 안에 관련 문제 3개 직접 풀이
- 일주일 안에 틀린 문제 원인 분류: 개념/계산/해석/주의력
- 다음 주 시작 전에 약점 1개만 명확히 보강

이 루프는 공부 시간을 늘리기보다 유지율을 높입니다. 2학년에서 실력이 갈리는 이유는 천재성보다 1학년 루틴의 안정성인 경우가 많습니다.

## 1학년 말 점검표

학년이 끝날 때 아래 항목을 기준으로 스스로 점검하면 이후 과목의 난이도를 크게 줄일 수 있습니다.

- 함수, 반복, 자료형, 예외 처리의 차이를 말로 설명할 수 있는가.
- 시간 복잡도의 기본 개념을 문제 풀이에 적용할 수 있는가.
- 행렬 곱의 의미를 계산 절차가 아니라 구조 관점으로 설명할 수 있는가.
- 논리식의 참/거짓을 코드 분기와 연결해 설명할 수 있는가.
- 혼자 푼 문제와 협업으로 푼 문제의 차이를 기록해 두었는가.

이 다섯 개가 채워지면 2학년 전공의 진입 비용이 크게 낮아집니다.

## 첫해에 실력을 벌리는 루틴 설계

1학년에서 실력이 벌어지는 지점은 난이도 높은 문제를 얼마나 빨리 푸느냐가 아닙니다. 기초 루틴을 얼마나 안정적으로 반복하느냐입니다. 권장 루틴은 단순합니다. 강의 당일 20분 요약, 이틀 내 40분 복습, 주말 60분 오답 정리입니다. 이 세 단계를 한 과목이라도 8주 이상 유지하면 중간고사 이후에도 이해도가 급격히 떨어지지 않습니다.

또한 첫해에는 결과 중심 목표보다 과정 중심 목표를 함께 세워야 합니다. 예를 들어 "A 학점"만 목표로 두면 시험 이후 기억이 빠르게 사라질 수 있습니다. 반면 "함수와 반복문을 빈 종이에 구현", "행렬 연산 의미를 말로 설명", "논리식 오류를 코드 반례로 찾기" 같은 목표를 두면 다음 학년 연결 비용이 줄어듭니다.

## 실행 가능한 학습 운영 원칙

아래 원칙은 전공 주제가 달라도 공통으로 적용됩니다. 첫째, 매주 하나의 "관찰 가능한 결과"를 남겨야 합니다. 결과물은 코드, 표, 짧은 리포트, 테스트 로그 중 무엇이든 좋지만 타인이 재확인할 수 있어야 합니다. 둘째, 선택 이유를 한 문장으로 기록해야 합니다. 같은 결과가 나와도 선택 근거가 없으면 다음 학습에서 재사용하기 어렵습니다. 셋째, 실패를 유형화해야 합니다. 단순히 틀렸다고 적지 말고 개념 혼동, 조건 누락, 검증 부족, 범위 과대 같은 형태로 분류하면 개선 속도가 빨라집니다.

또한 전공 학습은 "입력-처리-출력-회고"의 폐루프로 운영하는 편이 안정적입니다. 입력은 강의와 자료 조사, 처리는 문제 풀이와 구현, 출력은 산출물, 회고는 개선 기록입니다. 많은 학생이 입력과 처리에만 시간을 쓰고 출력과 회고를 생략하는데, 이 경우 학습이 누적되지 않습니다. 반대로 작은 출력이라도 꾸준히 남기면 포트폴리오, 면접, 협업 문서까지 한 번에 연결됩니다.

실제 주간 운영 예시는 다음과 같습니다.

- 월~화: 개념 정리와 핵심 문제 2개 해결
- 수~목: 구현 또는 실험 1회, 측정 지표 기록
- 금: 결과 요약 10줄 작성, 다음 주 약점 1개 선택
- 주말: 약점 보강 60분 + 문서 업데이트

이 방식의 장점은 완벽주의를 줄이고 지속 가능성을 높인다는 점입니다. 전공 학습에서 중요한 것은 단기간 최대치보다 장기간 반복 가능성입니다. 특히 학기 후반에는 과제와 시험이 겹치므로, 미리 정해 둔 최소 루틴이 없으면 우선순위가 흔들립니다.

마지막으로, 스스로에게 던질 검증 질문을 고정해 두면 품질이 올라갑니다. "이번 주 결과를 타인이 10분 안에 재현할 수 있는가", "내 선택 이유를 반대 관점에서 반박해도 버틸 수 있는가", "다음 주에 같은 문제를 더 짧게 풀 수 있는가" 같은 질문입니다. 이 질문에 답할 수 있다면 학습은 단순 반복을 넘어 실제 역량으로 축적되고 있다고 볼 수 있습니다.

## 마지막 보강: 학기 운영과 검증 습관

전공 학습을 장기적으로 유지하려면 학기 중간에 한 번, 학기 말에 한 번 구조 점검을 수행하는 것이 좋습니다. 중간 점검에서는 "진도는 따라가지만 이해가 빈약한 영역"을 찾고, 학기 말 점검에서는 "다음 과목으로 넘길 수 있는 수준인지"를 확인합니다. 이때 핵심은 감정 평가가 아니라 증거 평가입니다. 예를 들어 개념을 설명한 노트, 문제 풀이 로그, 실행 가능한 코드, 실험 결과 표, 회고 문서 같은 증거가 있으면 준비 상태를 명확히 판단할 수 있습니다.

또한 학기 운영에서 가장 흔한 실패는 우선순위 충돌입니다. 과제 마감과 시험 대비, 프로젝트 통합이 동시에 몰리면 쉽게 무너집니다. 이를 줄이려면 주간 계획에 반드시 완충 구간을 넣어야 합니다. 완충 구간은 비어 있는 시간이 아니라, 예상 밖 오류를 처리하는 시간입니다. 이 습관이 있으면 긴급 상황에서 전체 루틴이 붕괴하지 않습니다.

마지막으로 협업 과목이 있는 경우 개인 학습과 팀 산출물의 연결을 의도적으로 설계해야 합니다. 개인이 정리한 개념 노트와 팀의 의사결정 문서를 분리하지 말고, PR 설명이나 회의록에 근거를 남기는 방식으로 연결하면 학습과 결과가 동시에 강화됩니다. 이런 기록은 졸업 이후에도 포트폴리오, 면접 답변, 온보딩 문서로 재활용할 수 있는 장기 자산이 됩니다.

## 짧은 실천 과제

이번 주 안에 적용할 수 있는 가장 작은 과제는 세 가지입니다. 첫째, 본문에서 다룬 핵심 개념 하나를 5문장으로 요약합니다. 둘째, 해당 개념이 실제 코드 또는 과제에서 어떻게 나타나는지 예시를 1개 적습니다. 셋째, 이해가 부족한 지점을 질문 형태로 1개 남깁니다. 이 세 단계는 분량이 작지만 학습의 방향을 잃지 않게 해 주며, 다음 학습 행동을 구체적으로 만들어 줍니다.

중요한 점은 완벽한 답안을 만드는 것이 아니라, 다음 반복에서 개선 가능한 초안을 남기는 것입니다. 반복 가능한 초안이 쌓일수록 전공 학습은 단발성 이벤트가 아니라 누적형 성장으로 전환됩니다.

## 마무리 보완 메모

학습 계획은 길게 적는 것보다 실행 가능하게 적는 편이 더 효과적입니다. 이번 단원에서는 개념 이해, 예시 적용, 질문 정리를 하나의 반복 루프로 묶는 것이 핵심입니다. 특히 다음 과목으로 넘어가기 전에 최소 한 번은 본문 핵심을 자신의 언어로 다시 설명해 보고, 실제 과제나 코드에서 동일 개념을 확인해 두면 전이 학습이 쉬워집니다. 작은 반복을 꾸준히 수행하는 습관이 결국 큰 격차를 만듭니다.

## 1학년 학습 계획 템플릿

1학년은 속도보다 구조가 중요합니다. 아래 템플릿처럼 과목별 목표와 검증 방식을 같은 표에 두면 중간고사 이후에도 루틴이 무너지지 않습니다.

| 과목 | 주간 목표 | 검증 방식 | 보강 조건 |
| --- | --- | --- | --- |
| 미적분 | 핵심 정리 2개 + 문제 10개 | 오답 원인 3유형 분류 | 오답률 30% 초과 시 주말 90분 보강 |
| 선형대수 | 벡터/행렬 개념 요약 | 손계산 5문제 + 코드 1회 | 개념 설명 실패 시 노트 재작성 |
| 이산수학 | 명제/집합/증명 훈련 | 증명 문제 3개 구두 설명 | 논리 비약 발견 시 스터디 질문 등록 |
| 프로그래밍 | 함수/반복/예외 실습 | 테스트 통과 + 코드 리뷰 1회 | 테스트 실패 2회 이상 시 범위 축소 |

또한 일부 학과는 상대평가를 적용하므로 GPA 계산 규칙을 미리 이해해 두는 편이 유리합니다.

```text
예시 GPA = (A0 3학점×4.0 + B+ 3학점×3.5 + A+ 3학점×4.5 + B0 3학점×3.0) / 12
         = 45.0 / 12 = 3.75
```

학점 계산의 목적은 점수 집착이 아니라 위험 과목을 조기에 찾는 데 있습니다.

## 정리

1학년 과목은 전공다운 재미를 늦추는 구간이 아니라, 이후 과목을 버티게 만드는 기초 체력 구간입니다. 미적분, 선형대수, 이산수학, 프로그래밍 입문을 하나의 준비 운동으로 이해하면 학습 방향이 훨씬 선명해집니다. 다음 글에서는 많은 학생이 전공의 중심으로 느끼는 자료구조와 알고리즘을 봅니다.

## 처음 질문으로 돌아가기

- **왜 1학년 과목은 수학과 프로그래밍 비중이 유난히 클까요?**
  - 1학년 과목은 수학적 사고와 구현 습관을 동시에 고정하는 구간이기 때문입니다. 초반에 기초를 단단히 만들면 2학년 이후 과목의 이해 속도와 안정성이 크게 올라갑니다.
- **미적분, 선형대수, 이산수학, 프로그래밍 입문은 각각 어떤 역할을 맡을까요?**
  - 미적분은 변화량, 선형대수는 벡터와 행렬 구조, 이산수학은 논리와 증명, 프로그래밍 입문은 구현 기초를 담당합니다. 네 과목은 역할이 다르지만 함께 연결되어 후속 과목의 기반을 만듭니다.
- **지금은 진로와 멀어 보이는 과목이 왜 2학년 이후 성과를 가를까요?**
  - 2학년 이후 과목은 1학년 개념을 전제로 설계되어 있습니다. 기초가 약하면 같은 내용을 반복 복습하느라 시간이 소모되고 프로젝트에서도 구현 품질이 흔들립니다.
