# Computer Science Major 101 (2/10): 1학년 과목 이해하기

1학년 시간표를 보면 전공다운 이름보다 수학과 기초 과목이 먼저 눈에 들어옵니다. 코딩을 배우러 왔다고 생각한 학생에게는 이 구성이 다소 멀게 느껴질 수 있습니다.

이 글은 Computer Science Major 101 시리즈의 2번째 글입니다.

## 먼저 던지는 질문

- 왜 1학년 과목은 수학과 프로그래밍 비중이 유난히 클까요?
- 미적분, 선형대수, 이산수학, 프로그래밍 입문은 각각 어떤 역할을 맡을까요?
- 지금은 진로와 멀어 보이는 과목이 왜 2학년 이후 성과를 가를까요?

*Computer Science Major 101 2장 흐름 개요*

이 그림에서는 1학년 과목 이해하기를 운영 흐름 안에서 어디에 배치해야 하는지 봅니다. 핵심은 개념을 따로 외우는 것이 아니라 입력, 처리, 검증, 운영 신호가 어떤 경계로 이어지는지 확인하는 데 있습니다.

> 1학년 과목 이해하기의 핵심은 기능 이름이 아니라, 어떤 경계에서 무엇을 검증하고 어떤 신호를 남길지 정하는 데 있습니다.

## 이 글에서 배울 것

- 미적분의 의미
- 선형대수의 역할
- 이산수학의 쓰임
- 프로그래밍 입문 과목의 성격
- 공부 습관의 중요성

## 왜 중요한가

1학년 기초는 알고리즘, AI, 시스템처럼 뒤에 나오는 모든 과목의 바닥입니다. 이 시기에 쌓은 기초가 약하면 상위 과목에서 계속 같은 구멍을 다시 메우게 됩니다.

## 한눈에 보는 개념

> 1학년 과목은 나중에 다시 만날 개념을 미리 깔아 두는 기반 공사에 가깝습니다.

과목은 병렬로 들을 수 있어도 이해의 흐름은 서로 연결됩니다. 변화량을 다루는 힘, 벡터와 행렬 감각, 논리와 집합 사고, 기본 코딩 습관이 함께 쌓여야 다음 학년 과목이 버텨집니다.

## 핵심 용어

- **미적분(calculus)**: 연속적으로 변하는 값을 다루는 수학입니다.
- **선형대수(linear algebra)**: 벡터와 행렬을 중심으로 구조를 보는 수학입니다.
- **이산수학(discrete math)**: 논리와 집합, 그래프를 다루는 수학입니다.
- **프로그래밍 입문(intro programming)**: 언어와 프로그램 기본기를 배우는 과목입니다.
- **실습(lab)**: 손으로 직접 구현해 보는 시간입니다.

## Before/After

**Before**: 1학년 과목이 진로와 멀어 보입니다.

**After**: 이후 모든 전공 과목의 뿌리라는 점이 보입니다.

## 실습: 1학년 과목 매트릭스

### 1단계 — 과목 나열

```python
courses = ["calculus", "linalg", "discrete", "intro_prog"]
```

학교마다 과목명은 조금씩 달라도 1학년 핵심 축은 대체로 이 범위에서 크게 벗어나지 않습니다.

### 2단계 — 상위 과목 연결

```python
maps = {"calculus": "ml", "linalg": "ml", "discrete": "algorithms", "intro_prog": "all"}
```

각 과목이 어디로 이어지는지 적어 보면 목적이 분명해집니다. 특히 프로그래밍 입문은 사실상 모든 후속 과목의 공통 기반입니다.

### 3단계 — 주당 학습 시간

```python
hours = {c: 6 for c in courses}
```

기초 과목은 감으로 공부하면 쉽게 밀립니다. 주당 시간을 먼저 배정해 두면 체감 난이도에 따라 한 과목만 붙잡는 실수를 줄일 수 있습니다.

### 4단계 — 실습 비중

```python
labs = {"intro_prog": 4, "discrete": 1}
```

프로그래밍은 강의를 이해하는 것과 직접 손을 움직이는 것이 별개입니다. 실습 시간을 따로 확보하지 않으면 문법은 아는데 구현이 안 되는 상태가 오래 갑니다.

### 5단계 — 약점 표시

```python
weak = [c for c, h in hours.items() if h < 5]
```

약한 과목을 빨리 찾는 편이 유리합니다. 초반 약점은 시간이 갈수록 더 비싼 복습 비용으로 돌아오기 때문입니다.

## 이 코드에서 먼저 볼 점

- 각 과목은 상위 과목과 연결됩니다.
- 학습 시간은 곧 기초 체력입니다.
- 실습 시간은 별도로 계산해야 합니다.

## 자주 하는 실수 5가지

1. **출석만 하고 복습하지 않는 일입니다.**
2. **수학을 이해가 아니라 암기로만 버티는 일입니다.**
3. **언어를 자꾸 바꾸면서 기초를 미루는 일입니다.**
4. **실습 과제를 마감 직전에 시작하는 일입니다.**
5. **질문하지 않고 혼자 오래 끌고 가는 일입니다.**

## 실무에서는 이렇게 드러납니다

실무에서 바로 미적분 문제를 풀지는 않더라도, 논리와 구조가 흔들리면 코드 리뷰와 문제 해결 방식도 함께 흔들립니다. 상위 과목과 현업 역량은 생각보다 1학년 기초 위에 정직하게 서 있습니다.

## 선배 엔지니어는 이렇게 봅니다

- 기초는 일찍 쌓을수록 복리처럼 남습니다.
- 수학은 짧게 외우기보다 천천히 이해하는 편이 오래 갑니다.
- 코드는 매일 조금씩 보는 습관이 중요합니다.
- 실수는 실패가 아니라 학습 기록입니다.
- 질문은 초반에 붙일수록 강한 습관이 됩니다.

## 정리

1학년 과목은 전공다운 재미를 늦추는 구간이 아니라, 이후 과목을 버티게 만드는 기초 체력 구간입니다. 미적분, 선형대수, 이산수학, 프로그래밍 입문을 하나의 준비 운동으로 이해하면 학습 방향이 훨씬 선명해집니다. 다음 글에서는 많은 학생이 전공의 중심으로 느끼는 자료구조와 알고리즘을 살펴보겠습니다.
