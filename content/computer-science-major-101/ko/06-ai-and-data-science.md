# Computer Science Major 101 (6/10): AI와 데이터사이언스

AI와 데이터사이언스는 가장 화려해 보이는 분야이지만, 막상 들어가 보면 통계와 데이터 정리, 평가와 검증 같은 기초 작업이 훨씬 더 많은 비중을 차지합니다. 모델 이름만 외우는 방식으로는 오래 버티기 어렵습니다.

이 글은 Computer Science Major 101 시리즈의 6번째 글입니다.

## 먼저 던지는 질문

- AI와 데이터사이언스는 전공 안에서 어떻게 나뉘고 어떻게 연결될까요?
- 통계, 머신러닝, 딥러닝, 데이터 분석은 어떤 흐름으로 이어질까요?
- 모델만 보는 공부가 왜 금방 한계에 부딪힐까요?

> AI와 데이터사이언스의 핵심은 모델이나 기법이 아니라, 데이터의 가정과 문제 상황을 올바르게 짝지어 선택하는 데 있습니다.

## 이 글에서 배울 것

- 통계의 기초
- 머신러닝 입문
- 딥러닝의 기본 위치
- 데이터 분석
- 두 영역의 역할 분담

## 왜 중요한가

데이터 감각은 이제 특정 직무만의 능력이 아니라 거의 모든 현대 엔지니어에게 필요한 기본기입니다. 무엇보다 모델 성능은 데이터와 평가 체계 위에서만 의미를 가지므로, 겉으로 보이는 결과보다 바닥 구조를 읽는 힘이 더 중요합니다.

## 한눈에 보는 개념

> 데이터는 재료이고, 통계는 읽는 언어이며, 모델은 그 위에서 패턴을 계산하는 도구입니다.

데이터가 출발점이고, 통계는 그 데이터를 읽는 언어입니다. 머신러닝과 신경망은 그 위에서 예측 모델을 만들고, 분석은 결과를 해석하고 실제 문제에 연결합니다. 분야 이름이 달라도 흐름은 이어져 있습니다.

## 핵심 용어

- **특성(feature)**: 모델이 입력으로 받는 변수입니다.
- **레이블(label)**: 맞혀야 하는 목표 값입니다.
- **학습(training)**: 데이터로부터 패턴을 익히는 과정입니다.
- **추론(inference)**: 학습한 모델로 예측을 수행하는 과정입니다.
- **데이터셋(dataset)**: 분석과 학습에 쓰는 데이터 묶음입니다.

## Before/After

**Before**: 모델 이름과 성능 숫자만 봅니다.

**After**: 데이터 품질, 분포, 평가 방법까지 함께 봅니다.

## 실습: 미니 ML 파이프라인

### 1단계 — 데이터

```python
xs = [1, 2, 3, 4]
ys = [2, 4, 6, 8]
```

가장 작은 지도 학습 예시입니다. 입력과 정답의 짝이 있어야 이후 단계가 의미를 가집니다.

### 2단계 — 평균

```python
avg = sum(ys) / len(ys)
```

평균 같은 기초 통계량은 모델보다 먼저 데이터를 읽는 출발점입니다. 데이터의 중심도 모른 채 모델만 고르는 습관은 오래 가지 않습니다.

### 3단계 — 회귀 기울기

```python
def slope(xs, ys):
    mx, my = sum(xs)/len(xs), sum(ys)/len(ys)
    num = sum((x-mx)*(y-my) for x, y in zip(xs, ys))
    den = sum((x-mx)**2 for x in xs)
    return num / den
```

가장 단순한 회귀 예시지만, 모델도 결국 데이터 관계를 수학적으로 근사하는 함수라는 사실을 잘 보여 줍니다.

### 4단계 — 예측

```python
m = slope(xs, ys)
pred = m * 5
```

학습이 끝나면 새로운 입력에 값을 예측합니다. 이 단계가 현업에서는 추론 비용, 응답 속도, 운영 품질과 연결됩니다.

### 5단계 — 평가

```python
mae = sum(abs(m*x - y) for x, y in zip(xs, ys)) / len(xs)
```

모델은 결과를 내는 것만으로 충분하지 않습니다. 얼마나 틀렸는지 수치로 봐야 비로소 비교와 개선이 가능합니다.

## 이 코드에서 먼저 볼 점

- 통계는 모델보다 먼저 데이터를 읽게 해 줍니다.
- 예측은 결국 함수 적용입니다.
- 평가는 학습 결과를 믿어도 되는지 보여 주는 근거입니다.

## 자주 하는 실수 5가지

1. **데이터 누수를 만드는 일입니다.**
2. **학습 데이터와 테스트 데이터를 섞는 일입니다.**
3. **스케일 차이를 무시한 채 비교하는 일입니다.**
4. **기준선 모델 없이 복잡한 모델부터 고르는 일입니다.**
5. **평가 지표를 모호하게 두는 일입니다.**

## 실무에서는 이렇게 드러납니다

모델 운영에서 가장 중요한 능력 중 하나는 분포 변화 감지입니다. 학습할 때의 데이터와 운영 데이터가 달라지면 성능은 빠르게 흔들립니다. 그래서 강한 팀일수록 모델 자체보다 데이터 흐름과 평가 체계를 더 자주 점검합니다.

## 선배 엔지니어는 이렇게 봅니다

- 모델보다 데이터가 더 중요할 때가 많습니다.
- 복잡한 모델보다 기준선 모델이 먼저입니다.
- 평가는 주장보다 강한 증거입니다.
- 해석 가능성도 중요합니다.
- 재현성이 없으면 결과를 믿기 어렵습니다.

## 정리

AI와 데이터사이언스는 다른 이름을 갖고 있지만, 전공 안에서는 데이터 이해와 모델링이라는 하나의 흐름 안에 놓여 있습니다. 통계와 데이터 분석이 바닥을 만들고, 그 위에 머신러닝과 딥러닝이 올라갑니다. 다음 글에서는 이렇게 쌓은 지식을 실제 산출물로 묶는 프로젝트 과목을 살펴보겠습니다.

## AI 학습을 모델 이름 중심에서 문제 중심으로 전환하기

AI와 데이터사이언스 과목을 들을 때 가장 큰 함정은 최신 모델 이름을 많이 아는 것이 실력이라고 착각하는 일입니다. 실제 실력은 문제 정의, 데이터 수집/정제, 기준선 설정, 평가 설계, 오류 분석, 운영 모니터링까지 이어지는 전체 파이프라인을 일관되게 설명할 수 있는지로 판단됩니다. 모델은 이 파이프라인의 한 단계일 뿐입니다.

아래 표는 전공 단계에서 자주 만나는 분석 과제를 기준으로 접근법을 정리한 것입니다.

| 문제 유형 | 입력 데이터 | 1차 기준선 | 고도화 방향 | 핵심 지표 |
| --- | --- | --- | --- | --- |
| 이진 분류 | 고객 이탈 여부, 스팸 여부 | 로지스틱 회귀 | 트리 앙상블, 신경망 | Precision/Recall/F1 |
| 회귀 | 수요 예측, 가격 예측 | 선형 회귀 | GBM, 시계열 모델 | MAE/RMSE |
| 군집 | 사용자 세그먼트 | K-means | HDBSCAN, 임베딩 기반 군집 | 실루엣 점수, 해석 가능성 |
| 추천 | 사용자-아이템 로그 | 인기순 baseline | 협업 필터링, 딥러닝 | CTR, NDCG |

표의 목적은 특정 모델 추천이 아니라 "항상 기준선부터 시작"하는 습관입니다. 기준선이 없으면 개선 여부를 증명할 수 없습니다.

## 통계 과목과 ML 과목의 연결 지점

- 표본과 모집단 개념은 데이터 편향을 설명하는 출발점입니다.
- 분산과 공분산 이해는 특성 스케일링과 상관 구조 해석에 필요합니다.
- 가설검정 개념은 성능 차이가 우연인지 아닌지 판단하는 근거입니다.
- 과적합/과소적합 진단은 편향-분산 균형 감각과 직결됩니다.

즉, 통계는 ML을 위한 선택 과목이 아니라 안전장치입니다. 통계가 약하면 모델 성능 숫자를 과신하기 쉽고, 운영에서 성능 하락 원인을 잘못 해석하게 됩니다.

## 실전형 학습 루프

1. 문제를 비즈니스 문장으로 다시 쓰기
2. 라벨 정의와 평가 지표를 먼저 고정하기
3. 기준선 모델 구축 후 오차 사례 분석
4. 고도화 모델 도입 전 데이터 품질 점검
5. 결과를 재현 가능한 노트/리포트로 기록

이 루프를 한 번이라도 끝까지 돌려 보면 AI 과목이 도구 사용 과목이 아니라 판단 과목이라는 점이 분명해집니다.

## 모델 성능보다 평가 설계가 먼저인 이유

같은 정확도 수치라도 데이터 분포와 오류 비용이 다르면 의미가 달라집니다. 예를 들어 의료·보안처럼 거짓 음성 비용이 큰 영역에서는 정확도보다 재현율이 더 중요할 수 있습니다. 반대로 추천 시스템에서는 랭킹 지표가 체감 품질과 더 가깝습니다. 따라서 지표 선택은 기술 선택이 아니라 문제 정의의 일부입니다.

학부 단계에서는 "작은 재현 실험"을 반복하는 것이 좋습니다. 같은 데이터에서 특성 전처리만 바꾸고 결과 차이를 기록하거나, 기준선 대비 개선 폭을 표로 정리하는 방식입니다. 이 습관이 있으면 새 모델을 도입할 때도 과장 없이 판단할 수 있습니다.

## 실행 가능한 학습 운영 원칙

아래 원칙은 전공 주제가 달라도 공통으로 적용됩니다. 첫째, 매주 하나의 "관찰 가능한 결과"를 남겨야 합니다. 결과물은 코드, 표, 짧은 리포트, 테스트 로그 중 무엇이든 좋지만 타인이 재확인할 수 있어야 합니다. 둘째, 선택 이유를 한 문장으로 기록해야 합니다. 같은 결과가 나와도 선택 근거가 없으면 다음 학습에서 재사용하기 어렵습니다. 셋째, 실패를 유형화해야 합니다. 단순히 틀렸다고 적지 말고 개념 혼동, 조건 누락, 검증 부족, 범위 과대 같은 형태로 분류하면 개선 속도가 빨라집니다.

또한 전공 학습은 "입력-처리-출력-회고"의 폐루프로 운영하는 편이 안정적입니다. 입력은 강의와 자료 조사, 처리는 문제 풀이와 구현, 출력은 산출물, 회고는 개선 기록입니다. 많은 학생이 입력과 처리에만 시간을 쓰고 출력과 회고를 생략하는데, 이 경우 학습이 누적되지 않습니다. 반대로 작은 출력이라도 꾸준히 남기면 포트폴리오, 면접, 협업 문서까지 한 번에 연결됩니다.

실제 주간 운영 예시는 다음과 같습니다.

- 월~화: 개념 정리와 핵심 문제 2개 해결
- 수~목: 구현 또는 실험 1회, 측정 지표 기록
- 금: 결과 요약 10줄 작성, 다음 주 약점 1개 선택
- 주말: 약점 보강 60분 + 문서 업데이트

이 방식의 장점은 완벽주의를 줄이고 지속 가능성을 높인다는 점입니다. 전공 학습에서 중요한 것은 단기간 최대치보다 장기간 반복 가능성입니다. 특히 학기 후반에는 과제와 시험이 겹치므로, 미리 정해 둔 최소 루틴이 없으면 우선순위가 흔들립니다.

마지막으로, 스스로에게 던질 검증 질문을 고정해 두면 품질이 올라갑니다. "이번 주 결과를 타인이 10분 안에 재현할 수 있는가", "내 선택 이유를 반대 관점에서 반박해도 버틸 수 있는가", "다음 주에 같은 문제를 더 짧게 풀 수 있는가" 같은 질문입니다. 이 질문에 답할 수 있다면 학습은 단순 반복을 넘어 실제 역량으로 축적되고 있다고 볼 수 있습니다.

## 마지막 보강: 학기 운영과 검증 습관

전공 학습을 장기적으로 유지하려면 학기 중간에 한 번, 학기 말에 한 번 구조 점검을 수행하는 것이 좋습니다. 중간 점검에서는 "진도는 따라가지만 이해가 빈약한 영역"을 찾고, 학기 말 점검에서는 "다음 과목으로 넘길 수 있는 수준인지"를 확인합니다. 이때 핵심은 감정 평가가 아니라 증거 평가입니다. 예를 들어 개념을 설명한 노트, 문제 풀이 로그, 실행 가능한 코드, 실험 결과 표, 회고 문서 같은 증거가 있으면 준비 상태를 명확히 판단할 수 있습니다.

또한 학기 운영에서 가장 흔한 실패는 우선순위 충돌입니다. 과제 마감과 시험 대비, 프로젝트 통합이 동시에 몰리면 쉽게 무너집니다. 이를 줄이려면 주간 계획에 반드시 완충 구간을 넣어야 합니다. 완충 구간은 비어 있는 시간이 아니라, 예상 밖 오류를 처리하는 시간입니다. 이 습관이 있으면 긴급 상황에서 전체 루틴이 붕괴하지 않습니다.

마지막으로 협업 과목이 있는 경우 개인 학습과 팀 산출물의 연결을 의도적으로 설계해야 합니다. 개인이 정리한 개념 노트와 팀의 의사결정 문서를 분리하지 말고, PR 설명이나 회의록에 근거를 남기는 방식으로 연결하면 학습과 결과가 동시에 강화됩니다. 이런 기록은 졸업 이후에도 포트폴리오, 면접 답변, 온보딩 문서로 재활용할 수 있는 장기 자산이 됩니다.

## 짧은 실천 과제

이번 주 안에 적용할 수 있는 가장 작은 과제는 세 가지입니다. 첫째, 본문에서 다룬 핵심 개념 하나를 5문장으로 요약합니다. 둘째, 해당 개념이 실제 코드 또는 과제에서 어떻게 나타나는지 예시를 1개 적습니다. 셋째, 이해가 부족한 지점을 질문 형태로 1개 남깁니다. 이 세 단계는 분량이 작지만 학습의 방향을 잃지 않게 해 주며, 다음 학습 행동을 구체적으로 만들어 줍니다.

중요한 점은 완벽한 답안을 만드는 것이 아니라, 다음 반복에서 개선 가능한 초안을 남기는 것입니다. 반복 가능한 초안이 쌓일수록 전공 학습은 단발성 이벤트가 아니라 누적형 성장으로 전환됩니다.


## 마무리 보완 메모

학습 계획은 길게 적는 것보다 실행 가능하게 적는 편이 더 효과적입니다. 이번 단원에서는 개념 이해, 예시 적용, 질문 정리를 하나의 반복 루프로 묶는 것이 핵심입니다. 특히 다음 과목으로 넘어가기 전에 최소 한 번은 본문 핵심을 자신의 언어로 다시 설명해 보고, 실제 과제나 코드에서 동일 개념을 확인해 두면 전이 학습이 쉬워집니다. 작은 반복을 꾸준히 수행하는 습관이 결국 큰 격차를 만듭니다.


간단한 확인 항목으로는 개념을 말로 설명하기, 예시를 코드로 재현하기, 그리고 측정 결과를 한 줄로 남기기가 있습니다. 이 세 가지를 완료하면 단원 이해가 기억이 아니라 실행 수준으로 전환됩니다.


추가로, 모델 비교에서는 같은 검증 분할과 동일한 전처리 조건을 유지해야 결과 해석이 왜곡되지 않습니다. 실험 표를 남겨 재현성을 확보하는 습관이 실무 전환의 핵심입니다.


핵심은 비교 가능성과 재현 가능성입니다.

## 처음 질문으로 돌아가기

- **AI와 데이터사이언스는 전공 안에서 어떻게 나뉘고 어떻게 연결될까요?**
  - AI 모델은 데이터와 통계 위에 최적화 이론이 더해지며, 데이터사이언스는 모델뿐 아니라 문제 정의부터 결과 해석까지 전체 사이클을 다룹니다. 이 둘은 겹치기도 하고 구분되기도 합니다.
- **통계, 머신러닝, 딥러닝, 데이터 분석은 어떤 흐름으로 이어질까요?**
  - 선형회귀부터 신경망까지 모델의 계층이 올라갈수록 복잡도가 늘어나지만, 동시에 해석 어려움도 커집니다. 문제에 맞는 단순한 모델과 복잡한 모델을 구분할 수 있는 역량이 중요합니다.
- **모델만 보는 공부가 왜 금방 한계에 부딪힐까요?**
  - 실전 데이터는 교과서의 깔끔한 데이터와 다릅니다. 누락된 값, 이상치, 클래스 불균형 같은 현실을 다루고, 모델의 성능이 우연인지 유의미한 개선인지 검증할 수 있는 능력이 실무 역량입니다.
