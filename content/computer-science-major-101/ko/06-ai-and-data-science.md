# AI와 데이터사이언스

> 컴퓨터학과 전공 학습 가이드 101 시리즈 (6/10)

## 이 글에서 다룰 문제

- AI와 데이터사이언스는 같은 말처럼 쓰이지만 전공에서는 어떻게 나뉠까요?
- 통계, 머신러닝, 딥러닝, 데이터 분석은 어떤 순서로 이어질까요?
- 모델만 보는 공부가 왜 금방 한계에 부딪힐까요?
- 데이터 품질과 분포를 본다는 말은 실제로 무엇을 뜻할까요?

AI와 데이터사이언스는 요즘 가장 관심을 많이 받는 영역입니다. 그래서 처음 전공을 접하는 학생은 이 분야를 대단한 최신 기술 묶음처럼만 받아들이기 쉽습니다. 하지만 막상 수업을 듣기 시작하면 생각보다 기초 과목이 많이 깔려 있습니다. 통계, 선형대수, 프로그래밍, 데이터 처리, 평가 지표 같은 주제가 빠지지 않습니다.

이 영역을 이해할 때 가장 중요한 점은 **모델만 보지 않는 것**입니다. 좋은 모델은 좋은 데이터, 적절한 문제 정의, 명확한 평가 기준 위에서만 의미를 가집니다. 데이터사이언스는 데이터를 읽고 해석하는 힘에 가깝고, AI는 그 데이터로 예측하거나 판단하는 모델을 만드는 쪽에 더 가깝습니다.

이 글에서는 두 분야가 어떻게 연결되고 어디에서 갈라지는지, 전공 안에서는 어떤 과목 흐름으로 배우는지 정리하겠습니다.

## 이 글에서 배울 것

- 통계가 AI와 데이터사이언스의 바닥이 되는 이유
- 머신러닝과 딥러닝의 기본 위치
- 데이터 분석과 모델링의 역할 차이
- 평가와 재현성이 중요한 이유

## 왜 중요한가

현대 소프트웨어는 데이터 없이 움직이기 어렵습니다. 검색, 추천, 이상 탐지, 자동화, 예측, 운영 분석까지 거의 모든 영역에서 데이터 해석 능력이 필요합니다. 그래서 AI를 직접 하지 않더라도 **데이터 감각**은 많은 엔지니어에게 기본기가 되었습니다.

또한 이 분야는 겉보기에 화려하지만, 실제로는 실험 설계와 검증이 매우 중요합니다. 성능이 높아 보이는 모델도 데이터 누수나 잘못된 평가 때문에 쉽게 착시를 만들 수 있습니다. 그래서 통계와 평가를 함께 보는 훈련이 꼭 필요합니다.

## 한눈에 보는 흐름

```mermaid
flowchart LR
    D[데이터] --> S[통계]
    S --> M[머신러닝]
    M --> N[신경망]
    N --> A[분석]
```

데이터를 모으고 이해하는 일이 출발점입니다. 통계는 데이터를 읽는 언어를 제공하고, 머신러닝은 그 데이터를 학습 가능한 형태로 바꾸며, 신경망은 더 복잡한 모델링을 맡습니다. 마지막 분석 단계에서는 모델 결과를 해석하고 실제 문제에 붙이는 일이 중요해집니다.

## 핵심 용어

- **특성(feature)**: 모델이 입력으로 받는 변수입니다.
- **레이블(label)**: 예측하려는 정답 값입니다.
- **학습(training)**: 데이터로부터 패턴을 익히는 과정입니다.
- **추론(inference)**: 학습한 모델로 예측을 수행하는 과정입니다.
- **데이터셋(dataset)**: 분석과 학습에 사용하는 데이터 묶음입니다.

## Before/After

**Before**: 모델 이름과 성능 숫자만 봅니다.

**After**: 데이터 품질과 분포, 평가 방법까지 함께 봅니다.

## AI와 데이터사이언스는 역할이 조금 다릅니다

데이터사이언스 쪽에서는 데이터를 모으고 정리하고 시각화하고 해석하는 일이 큰 비중을 차지합니다. 어떤 분포를 갖는지, 이상치가 있는지, 결측치가 많은지, 어떤 특성이 중요한지 확인해야 합니다. 문제 정의가 틀리면 이후 모델링도 흔들립니다.

AI 쪽에서는 머신러닝과 딥러닝 모델을 설계하고 학습하고 비교하는 일이 더 앞에 옵니다. 하지만 여기서도 데이터가 부실하면 모델은 금방 무너집니다. 그래서 전공에서는 통계와 데이터 분석을 먼저 익히고, 그 위에 모델링을 얹는 흐름이 자연스럽습니다.

현업에서는 두 역할이 더 자주 섞입니다. 작은 팀에서는 한 사람이 데이터 정제, 실험, 모델 개선, 결과 해석까지 모두 맡는 경우도 많습니다. 그래서 분야 이름보다 기본 원리를 이해하는 편이 훨씬 오래 갑니다.

## 미니 ML 파이프라인

### 1단계 — 데이터

```python
xs = [1, 2, 3, 4]
ys = [2, 4, 6, 8]
```

가장 작은 형태의 학습 데이터입니다. 입력과 정답이 짝을 이룹니다.

### 2단계 — 평균

```python
avg = sum(ys) / len(ys)
```

평균 같은 기초 통계량은 데이터를 이해하는 첫 단계입니다. 모델링 전에 데이터의 중심과 범위를 보는 습관이 필요합니다.

### 3단계 — 회귀 기울기

```python
def slope(xs, ys):
    mx, my = sum(xs)/len(xs), sum(ys)/len(ys)
    num = sum((x-mx)*(y-my) for x, y in zip(xs, ys))
    den = sum((x-mx)**2 for x in xs)
    return num / den
```

가장 단순한 선형 회귀 감각을 보여 주는 코드입니다. 모델도 결국 수학적 관계를 근사하는 함수라는 점을 드러냅니다.

### 4단계 — 예측

```python
m = slope(xs, ys)
pred = m * 5
```

학습이 끝나면 새로운 입력에 대한 예측을 수행합니다. 이 단계가 추론입니다.

### 5단계 — 평가

```python
mae = sum(abs(m*x - y) for x, y in zip(xs, ys)) / len(xs)
```

예측이 얼마나 틀렸는지 계산합니다. 모델은 숫자만으로 끝나지 않고, 반드시 평가 기준으로 검증해야 합니다.

## 이 코드에서 주목할 점

- 통계는 모델보다 먼저 데이터를 읽게 해 줍니다.
- 예측은 결국 함수 적용이라는 점을 보여 줍니다.
- 평가는 학습 결과를 믿어도 되는지 판단하는 근거입니다.

## 자주 하는 실수 5가지

1. 데이터 누수를 만들어 놓고도 눈치채지 못하는 일입니다.
2. 학습 데이터와 테스트 데이터를 섞는 일입니다.
3. 스케일 차이를 무시한 채 성능을 비교하는 일입니다.
4. 기준선 모델 없이 복잡한 모델부터 붙잡는 일입니다.
5. 평가 지표를 모호하게 두는 일입니다.

## 실무에서는 이렇게 쓰입니다

모델 운영에서는 분포 변화 감지가 매우 중요합니다. 학습할 때 쓰인 데이터와 실제 운영 데이터가 달라지면 성능이 쉽게 흔들리기 때문입니다. 그래서 좋은 팀은 모델 자체보다 데이터 흐름과 평가 체계를 더 자주 점검합니다.

## 선배 엔지니어는 이렇게 봅니다

- 모델보다 데이터가 더 중요할 때가 많습니다.
- 복잡한 모델보다 기준선 모델이 먼저입니다.
- 평가는 주장보다 강한 증거입니다.
- 해석 가능성도 성능만큼 중요할 수 있습니다.
- 재현성이 없으면 결과를 신뢰하기 어렵습니다.

## 정리 및 다음 단계

AI와 데이터사이언스는 서로 다른 이름을 갖고 있지만, 전공 안에서는 데이터 이해와 모델링이라는 연속선 위에 놓여 있습니다. 통계와 데이터 분석이 바닥을 만들고 그 위에 머신러닝과 딥러닝이 올라갑니다. 다음 글에서는 이렇게 쌓은 지식을 실제 결과물로 묶는 프로젝트 과목을 보겠습니다.
